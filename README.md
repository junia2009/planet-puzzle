# 🪐 PLANET PUZZLE

宇宙をテーマにしたマッチ3パズルゲームです。重なり合う惑星タイルをタップして同じ種類を3枚揃えて消していき、すべてのタイルを消せばクリアです。

**▶ プレイはこちら：** https://junia2009.github.io/planet-puzzle/

## ゲーム仕様

### 基本ルール

| 項目 | 内容 |
|------|------|
| **目的** | すべてのタイル（405枚）を消してクリア |
| **操作** | タイルをタップ/クリックしてスロットに送る |
| **マッチ条件** | スロットに同じ種類が **3枚** 揃うと自動消去 |
| **ゲームオーバー** | スロットが満杯（空きがない）になった時点 |

### タイルの種類

全部で **15種類**、それぞれ **27枚**（合計405枚）。

| 絵文字 | 名前 |
|--------|------|
| ☀️ | 太陽 |
| 🌕 | 月 |
| 🌍 | 地球 |
| 🪐 | 土星 |
| 🔴 | 火星 |
| 🔵 | 海王星 |
| 🟠 | 木星 |
| 🟡 | 金星 |
| ⚫ | 水星 |
| 🟣 | 天王星 |
| ⭐ | 星 |
| ☄️ | 彗星 |
| 🌙 | 三日月 |
| 🛸 | UFO |
| 💫 | 流れ星 |

### スキル

各スキルは **3回まで** 使用できます。

| スキル | アイコン | 効果 |
|--------|---------|------|
| **シャッフル** | 🔀 | ゲームエリア内タイルの種類をランダムに入れ替え（残り回数表示あり） |
| **元に戻す** | ↩️ | 直前のタイル操作を取り消す（残り回数表示あり） |

### ゲームフロー

```
タイトル画面
    ↓（GAME STARTをタップ）
ゲーム画面（タイルが405枚配置）
    ↓（タイルをタップ → スロットへ → 3枚揃えば消去を繰り返す）
    ├── タイルが0枚になれば → MISSION COMPLETE
    └── スロットが満杯になれば → ゲームオーバー
```

## 演出・エフェクト

### タイトル画面
- 🪐 惑星がゆっくり浮き沈みしながら回転
- 衛星（青・ピンク）が惑星の周りを周回
- 流れ星が画面を横切る（2.2秒ごと）
- **スターウォーズ風の宇宙船** がランダムに飛行（後述）

### ゲーム画面
- タイルをタップするとスロットの位置へ**飛翔アニメーション**で移動
- 3枚揃った瞬間、**スケールアップ＋フラッシュ後に縮小消去**するアニメーション

### クリア時（MISSION COMPLETE）
- 専用クリア画面がアニメーションで出現
- **紙吹雪 80枚**が降り注ぐ
- **星バースト 20個**が画面全体に出現
- 画面枠がグラデーションで光り流れる

### 宇宙船アニメーション（タイトル・ゲーム中 共通）

タイトル画面・ゲームプレイ中、背景を3種類の宇宙船がランダムに飛行します（4.5秒ごとにスポーン）。

| 宇宙船 | モデル | サイズ | 飛行角度 |
|--------|--------|--------|---------|
| **Xウイング** | スターウォーズ風戦闘機 | 58〜90px | ±20〜45度 |
| **TIEファイター** | スターウォーズ帝国軍機 | 36〜64px | ±30度 |
| **ミレニアムファルコン** | 白い円盤型貨物船 | 90〜120px | ±18度（ほぼ水平） |

すべてSVGインラインで描画し、Web Animations APIで動的に飛行座標を計算。機首が進行方向を向きます。

#### ミレニアムファルコン ハイパードライブ演出（レア・約18%）

ファルコン出現時、約18%の確率でハイパードライブ突入演出が発生します。

1. 画面中央付近まで通常飛行
2. **機体が瞬間消滅**
3. **青白いスピードライン9本** がバッと広がって0.9秒でフェードアウト

## BGM・サウンド

### 技術：Web Audio API

外部の音楽ファイルを使わず、**Web Audio API** でリアルタイムに音声を合成しています。オシレーター（音の発振器）を組み合わせて楽器音を近似しています。

```
音の流れ:
オシレーター（周波数を指定）
    → LPFフィルター（音を丸くする）
    → ゲイン（音量・エンベロープ制御）
    → リバーブ（残響）
    → マスターゲイン
    → スピーカー出力
```

### ゲームBGM：ヨーダのテーマ風

| レイヤー | 内容 |
|----------|------|
| **メロディ** | triangle + sine 二重奏、ビブラートLFO付き（弦楽器近似） |
| **ハープアルペジオ** | E/A/F#/B メジャーコードを8分音符刻みで上昇 |
| **ベースドローン** | E1/G#1/B1/E2 のコントラバス風持続音（LFOで揺らぎ） |
| **リバーブ** | 二段ディレイ(0.42s + 0.28s)でホール残響を近似 |

**BPM 92**、Eメジャー調。フレーズA（8小節）＋フレーズB（8小節）が無限ループします。

#### 曲を鳴らす仕組み

音符を `[周波数, 長さms, 音量]` の配列で定義し、累積時間でスケジューリングします。

```js
const melody = [
  [E3,  Q,  0.58],  // ミ、四分音符
  [Fs3, E,  0.54],  // ファ#、八分音符
  [A3,  DQ, 0.68],  // ラ、付点四分
  [0,   Q,  0   ],  // 休符
];

let t = 0;
melody.forEach(([freq, durMs, vol]) => {
  playNote(freq, durMs, vol, now + t / 1000);
  t += durMs;
});
```

譜面（音符列）があれば任意の曲を再現できます。ただしWeb Audio APIの制約上、音色はオシレーター波形の近似になります。

### タイトルBGM

ドローン＋コードパッド＋アルペジオの神秘的な宇宙系アンビエント。

### SFX（効果音）

| シーン | 音 |
|--------|-----|
| タイルタップ | 短い高音のポップ音 |
| マッチ消去 | 和音の上昇アルペジオ |
| ゲームオーバー | 低音の下降フレーズ |
| クリア | 明るい上昇ファンファーレ |

## 設計の解説（初学者向け）

### なぜ1ファイル（index.html）にすべて書いているの？

このゲームはサーバーが不要な **静的Webアプリ**です。HTML・CSS・JavaScriptを1ファイルにまとめることで、ファイルを開くだけでゲームが動きます。

- **HTML** → 画面の構造（タイトル画面・ゲームエリア・スロット・ボタン）
- **CSS** → 見た目のスタイル（色・アニメーション・レスポンシブ対応）
- **JavaScript** → ゲームのロジック（タイル生成・クリック処理・勝敗判定）

### PWA（Progressive Web App）とは？

| ファイル | 役割 |
|----------|------|
| `manifest.json` | アプリ名・アイコン・テーマカラーなどを定義 |
| `sw.js` | バックグラウンドで動くスクリプト。オフラインでも動くようキャッシュ管理 |

### タイルのデータ管理

```javascript
{
  id: 42,
  type: 3,        // 種類（0〜14）
  x: 120, y: 80,  // 座標
  layer: 3,       // レイヤー
  zIndex: 1000,
  elem: <div>,
  blocked: false  // 上に重なっているタイルがあるか
}
```

### ブロック判定の仕組み

```
タイルAがタイルBにブロックされる条件：
1. BのzIndex > AのzIndex
2. AとBの矩形が3px以上重なっている
```

### アンドゥ（元に戻す）の仕組み

スナップショット（スタック）方式。タイル操作のたびに全状態を保存し、元に戻す時に取り出します（LIFO）。3回まで使用可能。

### レスポンシブ対応

ゲームエリアは固定の `560×700px` ですが、CSS `transform: scale()` で縮小対応しています。

## ゲームロジック詳細

### タイル生成アルゴリズム

```
1. pool（405枚）を作成：15種類 × 27枚
2. pool をFisher-Yatesでシャッフル
3. 左端パイルを生成：2本×12枚（縦に12px間隔）
4. 残りをMahjong式7レイヤーに分配
   - 偶数層: オフセット(0, 0)
   - 奇数層: オフセット(35px, 35px)
5. z-index = 100 + レイヤー × 300
```

### マッチチェック

```javascript
slots.filter(s => s === type).length >= 3 → 3枚を消去
```

### タイルの重なりシステム

- タイルは **12レイヤー**（層）で積み重なって配置される
- 上のタイルに **1ピクセル以上重なっている**タイルは「ブロック状態」になりタップ不可
- ブロックされたタイルは暗く表示（`brightness(0.45)`）される
- 上のタイルを取り除くと、下のタイルのブロックが解除される

#### 配置の仕組み

```
ゲームエリア: 560 × 700 px
左端パイル: 2本（各12枚、縦に12px間隔で積み重ね）
通常グリッド: 7列 × 9行（= 63セル）× 7レイヤー（偶数層・奇数層で偏差 35px）
タイルサイズ: 70px（整列配置、ずれなし）
z-index:      通常 = 100 + レイヤー × 300、パイル = 2000 − 深さ × 5
```

1. 270枚のタイルをシャッフル
2. 各タイルに 64セル内のランダムなセル位置を割り当て
3. インデックス順にレイヤー（奥→手前）を決定
4. z-index（前後の表示順）= `10 + レイヤー × 30 + ランダム(0〜10)`

## 設計の解説（初学者向け）

### なぜ1ファイル（index.html）にすべて書いているの？

このゲームはサーバーが不要な **静的Webアプリ**です。HTML・CSS・JavaScriptを1ファイルにまとめることで、ファイルを開くだけでゲームが動きます。GitHubのホスティング（GitHub Pages）でもそのまま公開できます。

- **HTML** → 画面の構造（タイトル画面・ゲームエリア・スロット・ボタン）
- **CSS** → 見た目のスタイル（色・アニメーション・レスポンシブ対応）
- **JavaScript** → ゲームのロジック（タイル生成・クリック処理・勝敗判定）

### PWA（Progressive Web App）とは？

PWAとは、Webサイトをスマートフォンのホーム画面にアプリとして追加できる仕組みです。

| ファイル | 役割 |
|----------|------|
| `manifest.json` | アプリ名・アイコン・テーマカラーなどを定義。「ホーム画面に追加」時に使われる |
| `sw.js`（Service Worker） | バックグラウンドで動くスクリプト。ファイルをキャッシュしてオフラインでも動くようにする |
| `apple-touch-icon` | iOSのホーム画面アイコン。Canvas APIで動的に描画してPNGとして設定 |

### タイルのデータ管理

各タイルは JavaScript のオブジェクトとして管理されています。

```javascript
// タイル1枚のデータ構造
{
  id: 42,            // タイルの一意なID（0から連番）
  type: 3,           // 種類（0～14、🪐なら3）
  x: 120,            // ゲームエリア内のX座標（左からの距離px）
  y: 80,             // ゲームエリア内のY座標（上からの距離px）
  layer: 3,          // 配置レイヤー（通常タイル 0～6、パイル 20）
  zIndex: 1000,      // CSS z-index値（前後の重なり順）
  elem: <div>,       // 実際のHTML要素への参照
  blocked: false     // 上にタイルが重なってタップできない状態かどうか
}
```

### ブロック判定の仕組み

「上のタイルが重なっているかどうか」は **矩形の重なり検出** で判定しています。

```
タイルAがタイルBにブロックされる条件：
1. BのzIndex > AのzIndex（Bが視覚的に手前にある）
   ※ zIndexが同じ場合は id が大きい方（後から追加）が手前
2. AとBの矩形領域が3px以上重なっている
```

```
矩形重なり検出（rectsOverlap 関数）:
    タイルAの右端 > タイルBの左端
 かつ タイルAの左端 < タイルBの右端
 かつ タイルAの下端 > タイルBの上端
 かつ タイルAの上端 < タイルBの下端
```

### アンドゥ（元に戻す）の仕組み

「元に戻す」機能は **スナップショット（スタック）方式**で実装されています。

```javascript
// タイルをタップするたびに現在の状態を保存
history.push({
  tiles: tiles.map(t => ({ ...t })),  // タイル全情報のコピー
  slots: [...slots],                   // スロット内容のコピー
  extraSlots                           // スロット拡張数のコピー
});

// 元に戻す時は最新の保存データを取り出す（LIFO: 後入れ先出し）
const snap = history.pop();
// → snapの内容でタイル・スロットを復元
```

### レスポンシブ対応

ゲームエリアは固定の `560 × 700px` ですが、画面幅が小さいスマートフォンでも表示できるよう、CSS `transform: scale()` で縮小しています。

```javascript
const availW = Math.min(560, window.innerWidth - 32);
const scale = availW / 560;  // 例: 390px幅のiPhoneなら 358/560 ≈ 0.64
gameArea.style.transform = `scale(${scale})`;
```

### Service Worker のキャッシュ戦略

`sw.js` は **Cache First（キャッシュ優先）** 戦略を採用しています。

```
1. ブラウザがリソースを要求
2. Service Worker がキャッシュを確認
   ├── キャッシュにあれば → キャッシュから即返却（高速・オフライン対応）
   └── キャッシュになければ → ネットワークから取得
```

ゲームを更新した際は `CACHE_NAME` のバージョン番号を上げることで古いキャッシュが削除され、新しいファイルが配信されます。

## ゲームロジック詳細

### タイル生成アルゴリズム

```
1. pool（405枚）を作成：和15種類 × 27枚
2. pool をシャッフル（Fisher-Yates アルゴリズム）
3. 左端パイルを生成：2本×12枚 = 24枚（縦に12px閣隔で重なる、z-index高、2000-深さ×5順）
4. 残り381枚をMahjong式7層に分配
   - 偶数層(0,2,4,6): オフセット(0, 0)
   - 奇数層(1,3,5) : オフセット(35px, 35px) → 上層タイル1枚が下層上最大4枚に跨がる
5. z-index = 100 + レイヤー × 300
6. 各層GRID_COLS×GRID_ROWS（9×7=63セル）をシャッフルしその先頭Nセルに配置
7. ゲームエリアに <div> タグとして追加
```

### マッチチェックアルゴリズム

タイルが1枚スロットに入るたびに全種類をチェックし、3枚揃った種類を消去します。

```javascript
TILE_TYPES.forEach((_, type) => {
  const count = slots.filter(s => s === type).length;
  if (count >= 3) {
    let removed = 0;
    slots = slots.filter(s => {
      if (s === type && removed < 3) { removed++; return false; }
      return true;  // ← 4枚目以降は残す（通常は3の倍数なので発生しない）
    });
  }
});
```

### Fisher-Yates シャッフル

配列をランダムに並び替える標準的なアルゴリズムです。後ろから順に、現在位置以前のランダムな位置と交換します。

```javascript
function shuffle(arr) {
  for (let i = arr.length - 1; i > 0; i--) {
    const j = randInt(0, i);
    [arr[i], arr[j]] = [arr[j], arr[i]];
  }
  return arr;
}
```
