# 🪐 PLANET PUZZLE

宇宙をテーマにしたマッチ3パズルゲームです。重なり合う惑星タイルをタップして同じ種類を3枚揃えて消していき、すべてのタイルを消せばクリアです。

**▶ プレイはこちら：** https://junia2009.github.io/planet-puzzle/

## ゲーム仕様

### 基本ルール

| 項目 | 内容 |
|------|------|
| **目的** | すべてのタイル（270枚）を消してクリア |
| **操作** | タイルをタップ/クリックしてスロットに送る |
| **マッチ条件** | スロットに同じ種類が **3枚** 揃うと自動消去 |
| **ゲームオーバー** | スロットが満杯（空きがない）になった時点 |

### タイルの種類

全部で **10種類**、それぞれ **27枚**（合計270枚）。3の倍数なので必ず3枚揃えられます。

| 絵文字 | 名前 | タイルの見た目 |
|--------|------|--------------|
| ☀️ | 太陽 | 淡い黄色〜白のグラデーション（黄色グロー付き） |
| 🌕 | 月 | グレーがかった黄白（クレーター風） |
| 🌍 | 地球 | 青緑のグラデーション |
| 🪐 | 土星 | ゴールド系のグラデーション |
| 🔴 | 火星 | 赤茶色のグラデーション |
| 🔵 | 海王星 | 青のグラデーション |
| 🟠 | 木星 | オレンジ縞模様 |
| 🟡 | 金星 | 淡い黄色のグラデーション |
| ⚫ | 水星 | ダークグレーのグラデーション |
| 🟣 | 天王星 | 青みがかった紫のグラデーション |

### タイルの重なりシステム

- タイルは **12レイヤー**（層）で積み重なって配置される
- 上のタイルに **1ピクセル以上重なっている**タイルは「ブロック状態」になりタップ不可
- ブロックされたタイルは暗く表示（`brightness(0.45)`）される
- 上のタイルを取り除くと、下のタイルのブロックが解除される

#### 配置の仕組み

```
ゲームエリア: 560 × 560 px
グリッド:     8列 × 8行（= 64セル）
セル間隔:     58px（タイルサイズ70pxより小さい → 隣接タイルが12px重なる）
ランダムずれ: ±2px（タイルが完全な格子にならないよう微妙にずらす）
レイヤー数:   12層（270枚を12層に分けて配置）
```

1. 270枚のタイルをシャッフル
2. 各タイルに 64セル内のランダムなセル位置を割り当て
3. インデックス順にレイヤー（奥→手前）を決定
4. z-index（前後の表示順）= `10 + レイヤー × 30 + ランダム(0〜10)`

### スロットエリア

| 項目 | 内容 |
|------|------|
| **基本スロット数** | 7枠 |
| **スロット拡張後** | 最大10枠（スキルで+3） |
| **動作** | タイルをタップするとスロット左から順に格納 |
| **消去タイミング** | 同じ種類が3枚揃った瞬間に自動消去 |
| **ゲームオーバー判定** | スロット満杯 かつ 次のタイルを入れる空きがない時 |

### スキル

3種類のスキルが使えます。各スキルは条件付きで **1回のみ**使用可能です。

| スキル | アイコン | 効果 | 使用条件 |
|--------|---------|------|---------|
| **シャッフル** | 🔀 | ゲームエリア内のタイルをランダムな位置に再配置 | いつでも使用可（何度でも） |
| **元に戻す** | ↩️ | 直前の操作（タイルを拾う）を取り消す | 操作後に有効化（何度でも） |
| **スロット拡張** | ➕ | スロットを3枠追加（7→10枠） | 1回のみ（使用後はグレーアウト） |

### ゲームフロー

```
タイトル画面
    ↓（GAME STARTをタップ）
ゲーム画面（タイルが270枚配置された状態でスタート）
    ↓（タイルをタップ → スロットへ → 3枚揃えば消去を繰り返す）
    ├── タイルが0枚になれば → MISSION COMPLETE（クリア）
    └── スロットが満杯になれば → ゲームオーバー
    ↓（タイトルに戻る）
タイトル画面
```

### 演出・エフェクト

#### タイトル画面
- 🪐 惑星がゆっくり浮き沈みしながら回転
- 衛星（青・ピンク）が惑星の周りを周回
- 流れ星が画面を横切る（2.2秒ごと）

#### クリア時（MISSION COMPLETE）
- 専用クリア画面がアニメーションで出現
- **紙吹雪 80枚**が降り注ぐ
- **星バースト 20個**（⭐✨🌟💫など）が画面全体に出現
- 画面枠がグラデーションで光り流れる

## 設計の解説（初学者向け）

### なぜ1ファイル（index.html）にすべて書いているの？

このゲームはサーバーが不要な **静的Webアプリ**です。HTML・CSS・JavaScriptを1ファイルにまとめることで、ファイルを開くだけでゲームが動きます。GitHubのホスティング（GitHub Pages）でもそのまま公開できます。

- **HTML** → 画面の構造（タイトル画面・ゲームエリア・スロット・ボタン）
- **CSS** → 見た目のスタイル（色・アニメーション・レスポンシブ対応）
- **JavaScript** → ゲームのロジック（タイル生成・クリック処理・勝敗判定）

### PWA（Progressive Web App）とは？

PWAとは、Webサイトをスマートフォンのホーム画面にアプリとして追加できる仕組みです。

| ファイル | 役割 |
|----------|------|
| `manifest.json` | アプリ名・アイコン・テーマカラーなどを定義。「ホーム画面に追加」時に使われる |
| `sw.js`（Service Worker） | バックグラウンドで動くスクリプト。ファイルをキャッシュしてオフラインでも動くようにする |
| `apple-touch-icon` | iOSのホーム画面アイコン。Canvas APIで動的に描画してPNGとして設定 |

### タイルのデータ管理

各タイルは JavaScript のオブジェクトとして管理されています。

```javascript
// タイル1枚のデータ構造
{
  id: 42,            // タイルの一意なID（0から連番）
  type: 3,           // 種類（0〜9、🪐なら3）
  x: 120,            // ゲームエリア内のX座標（左からの距離px）
  y: 80,             // ゲームエリア内のY座標（上からの距離px）
  layer: 6,          // 配置レイヤー（奥=0、手前=11）
  zIndex: 194,       // CSS z-index値（前後の重なり順）
  elem: <div>,       // 実際のHTML要素への参照
  blocked: false     // 上にタイルが重なってタップできない状態かどうか
}
```

### ブロック判定の仕組み

「上のタイルが重なっているかどうか」は **矩形の重なり検出** で判定しています。

```
タイルAがタイルBにブロックされる条件：
1. BのzIndex > AのzIndex（Bが視覚的に手前にある）
   ※ zIndexが同じ場合は id が大きい方（後から追加）が手前
2. AとBの矩形領域が3px以上重なっている
```

```
矩形重なり検出（rectsOverlap 関数）:
    タイルAの右端 > タイルBの左端
 かつ タイルAの左端 < タイルBの右端
 かつ タイルAの下端 > タイルBの上端
 かつ タイルAの上端 < タイルBの下端
```

### アンドゥ（元に戻す）の仕組み

「元に戻す」機能は **スナップショット（スタック）方式**で実装されています。

```javascript
// タイルをタップするたびに現在の状態を保存
history.push({
  tiles: tiles.map(t => ({ ...t })),  // タイル全情報のコピー
  slots: [...slots],                   // スロット内容のコピー
  extraSlots                           // スロット拡張数のコピー
});

// 元に戻す時は最新の保存データを取り出す（LIFO: 後入れ先出し）
const snap = history.pop();
// → snapの内容でタイル・スロットを復元
```

### レスポンシブ対応

ゲームエリアは固定の `560 × 560px` ですが、画面幅が小さいスマートフォンでも表示できるよう、CSS `transform: scale()` で縮小しています。

```javascript
const availW = Math.min(560, window.innerWidth - 32);
const scale = availW / 560;  // 例: 390px幅のiPhoneなら 358/560 ≈ 0.64
gameArea.style.transform = `scale(${scale})`;
```

### Service Worker のキャッシュ戦略

`sw.js` は **Cache First（キャッシュ優先）** 戦略を採用しています。

```
1. ブラウザがリソースを要求
2. Service Worker がキャッシュを確認
   ├── キャッシュにあれば → キャッシュから即返却（高速・オフライン対応）
   └── キャッシュになければ → ネットワークから取得
```

ゲームを更新した際は `CACHE_NAME` のバージョン番号を上げることで古いキャッシュが削除され、新しいファイルが配信されます。

## ゲームロジック詳細

### タイル生成アルゴリズム

```
1. pool（270枚）を作成：各10種類 × 27枚
2. pool をシャッフル（Fisher-Yates アルゴリズム）
3. 64セル（8×8グリッド）のインデックスを270個作り、シャッフル
4. pool[i] のタイルにセル[i % 270 → セル位置] を割り当て
5. タイルのレイヤー = Math.floor(i / (270 / 12)) → 0〜11
6. z-index = 10 + レイヤー × 30 + randInt(0, 10)
7. 位置 = セルの基準座標 ± 2px のランダムずれ
8. ゲームエリアに <div> タグとして追加
```

### マッチチェックアルゴリズム

タイルが1枚スロットに入るたびに全種類をチェックし、3枚揃った種類を消去します。

```javascript
TILE_TYPES.forEach((_, type) => {
  const count = slots.filter(s => s === type).length;
  if (count >= 3) {
    let removed = 0;
    slots = slots.filter(s => {
      if (s === type && removed < 3) { removed++; return false; }
      return true;  // ← 4枚目以降は残す（通常は3の倍数なので発生しない）
    });
  }
});
```

### Fisher-Yates シャッフル

配列をランダムに並び替える標準的なアルゴリズムです。後ろから順に、現在位置以前のランダムな位置と交換します。

```javascript
function shuffle(arr) {
  for (let i = arr.length - 1; i > 0; i--) {
    const j = randInt(0, i);
    [arr[i], arr[j]] = [arr[j], arr[i]];
  }
  return arr;
}
```
